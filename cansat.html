Data Transmission Strategy

Step 1: Image Capture and Storage

The ESP32-CAM captures images and stores them on the SD card. It also serves these images over a local Wi-Fi network, making them accessible through a web interface.

Learn More

Step 2: Data Exchange

The ESP32-CAM communicates with the ESP32 main module, informing it when images are successfully stored. The ESP32 main module sends JSON data, containing sensor readings, to the ESP32-CAM for storage on the SD card.

Learn More

Step 3: Data Transmission to Ground Station

The ESP32 main module transmits the collected and stored JSON data to the ground station using the NRF24L01+PA+LNA module. This ensures real-time monitoring of the CanSatâ€™s environment.

Data Processing Strategy

Step 1: Data Reception

The ground station receives data from the ESP32 main module via the NRF24L01+ module. The received JSON data includes sensor readings and status updates.

Learn More

Step 2: Data Transmission via Flask

The received JSON data is transmitted to the PC using a Flask server. Flask handles the incoming data, processes it, and prepares it for rendering in a web interface.

Learn More

Step 3: Web Interface Display

The processed JSON data is dynamically converted into HTML content and displayed on the web interface. This allows real-time visualization of sensor data and system status on the PC.
